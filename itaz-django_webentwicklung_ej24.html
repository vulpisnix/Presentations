Das Datenbankmodell

                    
                    class Poll(models.Model):
                        id # Eine 'einmalige' Zahl (wird automatisch hoch gezählt)
                        title # Der Umfragen Titel
                        description # Die Umfragen Beschreibung
                        created_at # Erstelldatum
                        created_by # Ersteller
                        start_date # Startdatum der Umfrage
                        end_date # Enddatum der Umfrage
                        max_participants # Maximale Teilnehmer
  • Ich war hauptsächlich für das Datenbankmodell zuständig
  • Unter Anderem für die Modelle, die für die Umfragen eine zentrale Rolle spielen
  • Das hier ist das Modell für eine Umfrage
  • Eine Umfrage hat:
  • einen Titel
  • eine optionale Beschreibung
  • Erstelldatum und Ersteller
  • ein Start- und Enddatum
  • Und man kann eine maximale Anzahl an Teilnehmern bestimmen
                    
                    class PollElementBase(models.Model):
                        MULTI_CHOICE  = 'MC'
                        RATING = 'RA'

                        ELEMENT_TYPE_CHOICES = [
                            (MULTI_CHOICE, 'Multiple Choice'),
                            (RATING, 'Rating'),
                        ]

                        title # Überschrift des Elements
                        sub_text # Fabian Fragen :)
                        order # Sortierung
                        element_type # Type des Elements z.B.: Multiple Choice
                        required # Fabian Fragen :)
  • Wie ihr später sehen werdet, haben wir zurzeit die Möglichkeit multiple choice fragen, und Bewertungen in die Umfragen einzufügen
  • Damit wir flexibel neue Fragetypen hinzufügen können, habe ich für grundlegende Eigenschaften einer Frage, eine Base klasse erstellt.
  • Sie enthält:
    den Titel der Frage Eine Beschreibung z.B. für Notizen oder genauere Erläuterungen der Frage Ein Index für Bestimmung der Reihenfolge Eine interne Bestimmung des Fragentyps, um Datenbankabfragen zu vereinfachen Und eine Möglichkeit, Fragen als erforderlich zu setzen 
                    
                    class ElementMultiChoice(PollElementBase):
                        id # Eine 'einmalige' Zahl (wird automatisch hoch gezählt)
                        poll # Das Umfrage Objekt
                        is_single_choice # Boolean ob Single Choice oder nicht
- Das ist die Spezifizierung für die Multiple Choice fragen.
- Wie man in der Klammer sehen kann, erbt diese Klasse von der Base Klasse
- Das heißt, alle Elemente der Base Klasse sind auch in dieser Klasse enthalten.
- Außerdem gibt es ein Verweis auf ein Umfrageobjekt und man kann die Auswahl von Antwortmöglichkeiten auf 1 begrenzen 
                    
                    class ElementMultiChoiceEntry(models.Model):
                        id # Eine 'einmalige' Zahl (wird automatisch hoch gezählt)
                        multi_choice_instance # Das MultiChoice Element
                        choice_text # Anzeigetext
                        order # Sortierung
Mit diesem Modell kann man für eine bestimmte multiple choice frage, Antwortmöglichkeiten erstellen
Es gibt:
Den Verweis auf die MC Frage
Den Anzeigetext
Und den Sortierungsindex
                    
                    class ElementRating(PollElementBase):
                        id # Eine 'einmalige' Zahl (wird automatisch hoch gezählt)
                        poll # Das Umfrageobjekt
                        min_rating = 1 # Kleinste Bewertungszahl
                        max_rating = 5 # Größte Bewertungszahl
- Bei dem Bewertungsfragetyp wird auf eine Umfrage verwiesen, und einen Mindest- und Maximalwert 
                    
                    class Participant(models.Model):
                        id # Eine 'einmalige' Zahl (wird automatisch hoch gezählt)
                        ip # Ip des Teilnehmers
                        sessionId # Django SessionId des Teilnehmers
Mithilfe von der IP-Adresse und Session ID wird ein einzigartiges Teilnehmerobjekt erstellt.
Dabei bleiben die Teilnehmer anonym 
                    
                    class PollResult(models.Model):
                        id # Eine 'einmalige' Zahl (wird automatisch hoch gezählt)
                        pollId # Poll Objekt
                        participant # Teilnehmer Objekt
                        json # Raw Daten
- Die Ergebnisse der Umfragen werden in der Datenbank als JSON-Objekt gespeichert, mit Verweisen auf die Umfrage und den Teilnehmer 
                    
                    class Contact(models.Model):
                        lastname # Nachname
                        firstname # Vorname
                        email # E-Mail
                        reference # Betreff
                        message # Nachricht

Grundidee

Eine Seite zur Veranschaulichung von Umfragenergebnissen

Wenn man Umfragen macht, möchte man die Ergebnisse auch sinnvoll interpretieren können.
Dafür haben wir eine Auswertungsseite entwickelt, welche die Gesamtanzahl aller Antwortmöglichkeiten ausgibt

Herausforderungen

  • Großer Umfang
  • Berechnung von Ergebnissen
  • Darstellung von Ergebnissen
  • Sonst nichts 👍
Einige Herausforderungen die ICH dabei hatte, sind:
Ein großer Umfang an Funktionen
Die Berechnung von den Ergebnissen brauchte viele verschachtelte Schleifen, was den Code komplizierter gemacht hat
Für die Darstellung der Ergebnisse musste ich eine eigene JSON-Objektstruktur entwickeln
Aber die gute Nachricht ist, es gab nichts, was ich nicht beseitigen konnte.

Umsetzung

  • Auslagerung von Code in Helper Funktionen
  • Generierung von Fake Daten
  • Visualisierung von Daten
Um den Code lesbarer, wartbarer und wiederverwendbar zu machen, habe ich eine eigene Helper Klasse für die Verarbeitung der Ergebnisse entwickelt.
Außerdem, habe ich eine Seite zur Generierung von Testdaten erstellt. Das ist aufwendiger als man sich vielleicht denken kann.
Und zuletzt, habe ich die dynamische Darstellung der Ergebnisse implementiert
